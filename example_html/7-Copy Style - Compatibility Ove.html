Client compatibility matrix

This matrix records what mxd supports today, what it partially supports, and what remains planned. Every "Supported" or "Partial" claim cites the automated tests that back it. If a behaviour is not tested, it is not claimed.

Test suite location: All compatibility tests live in tests/ at repository root. Each client profile has a dedicated test module. Run with cargo test --test client_compat

Status definitions

Supported

All documented protocol features work. Automated tests verify handshake, login, file operations, chat, and news. No known regressions.

Partial

Core features verified. Specific transactions or edge cases remain untested. The matrix notes what is not yet confirmed.

Planned

Scheduled for future verification. The roadmap specifies which release milestone will address this client profile.

Verification strategy

Each supported client profile has a dedicated test suite that exercises the full protocol stack. Tests run against a real mxd instance with an in-memory SQLite database. The test client implements the Hotline wire protocol directly, not through a third-party library.

Test coverage requirements

For a client to be marked "Supported", tests must verify:

  • Handshake preamble exchange (protocol ID, version negotiation)
  • Login transaction (0x006B) with password verification
  • File listing and download (transaction 0x00C8, 0x00CA)
  • Chat message send and receive (transaction 0x0069)
  • XOR text encoding detection and handling (if client uses it)
  • Multi-fragment reassembly for large payloads

QA enforcement: The release checklist requires that all "Supported" claims in this matrix have passing tests in CI. If a test fails, the status downgrades to "Partial" or the client is removed from the matrix until the issue is resolved.

Release note compliance

Release notes must reference this matrix. The QA sign-off checklist in the repository enforces this. If you're reading a release note that claims support for a client profile not listed here as Supported, something has gone wrong.
