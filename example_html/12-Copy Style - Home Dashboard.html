01 — Overview

The Hotline protocol, treated seriously

mxd implements Hotline server protocol v1.8.5: login, chat, threaded news, hierarchical file sharing, and private messaging. It speaks the binary wire format — 20-byte frame headers, big-endian integers, multi-fragment reassembly — so vintage Hotline clients and modern alternatives like SynHX connect without shims or translation layers.

The name is a train reference. The architecture is not.

02 — Architecture

Built to be reasoned about

Hx

Hexagonal Architecture

Domain logic sits behind ports. The Wireframe transport adapter handles framing and routing; Diesel handles persistence. Neither leaks into the other. Swap the database. Swap the transport. The domain doesn't notice.

Db

Dual Database

SQLite for a single-file deployment you can carry in your pocket. PostgreSQL when you need concurrent access at scale. One codebase, dual migration trees, compile-time backend selection. Recursive CTEs work on both.

Cx

Client Compatibility

Hotline 1.8.5, Hotline 1.9, and SynHX. Banner field augmentation, XOR text obfuscation detection, sub-version gating — all handled at the adapter boundary. A compatibility matrix tracks what works, what's partial, and what remains roadmap work. No over-claiming.

Kn

Verified Correct

TLA+ specifications model the handshake state machine. Stateright explores concurrent session interleavings. Kani proves bounded invariants for XOR round-trips and frame validation. Protocol correctness is not left to hope.

03 — The Wire

Every byte accounted for

The Hotline transaction frame is 20 bytes: flags, reply bit, transaction type, ID, error code, total size, data size. When data_size < total_size, continuation frames follow until the payload is fully assembled. Each continuation must echo the original header's flags, type, ID, error, and total size.

mxd's codec validates all of this in a single decoding pass. Payloads exceeding 1 MiB are rejected. Fragment sizes are clamped to 32 KiB. Outbound encoding produces byte-for-byte parity with the legacy runtime.

04 — Running it

Configuration without ceremony

CLI flags, environment variables, and TOML config files. OrthoConfig merges them in that order: defaults < file < env < CLI. Set MXD_DATABASE to a Postgres connection string. Set MXD_BIND to a socket address. Drop a .mxd.toml for persistent defaults. Run mxd create-user to provision accounts. Migrations apply automatically.

systemd integration includes sd_notify readiness signalling, watchdog heartbeats, and structured journald logging with per-connection trace spans. Socket activation is supported for zero-downtime restarts.

05 — Confidence

Tested at every layer

Unit tests with rstest fixtures. BDD scenarios in Gherkin via rstest-bdd. Property tests with proptest. Integration tests that start the wireframe binary, complete the handshake over TCP, and exercise login, file listing, and news flows. Expect-based protocol conformance tests against the SynHX client binary.

AFL++ fuzzes parse_transaction nightly in CI. Crashes are uploaded as artefacts and become regression tests. The seed corpus is generated from protocol examples.

Embedded PostgreSQL test fixtures spin up isolated clusters per test. No external database required. No root access required.
