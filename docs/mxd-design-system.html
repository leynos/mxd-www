00 — Philosophy

Protocol Souk

The design language takes its name from the intersection of two worlds: the geometric precision of binary protocol specifications and the pattern-rich warmth of Moroccan architectural craft. Every surface in a souk tells a story of material and method. Every byte in a transaction frame carries meaning through position.

This is not retro nostalgia. It is the recognition that the Hotline protocol — with its big-endian integers, its XOR-obfuscated text fields, its 20-byte frame headers — is an artefact worthy of the same care given to zellij tilework: each piece cut precisely, placed deliberately, forming something greater than its parts.

Design principle: Warmth and precision are not opposites. The terracotta of Marrakesh walls and the cold logic of 0x006B (Login) coexist. The design language renders technical depth in materials that feel human.

02 — Typography

Three Voices

The type system uses three families, each with a distinct voice. Together they allow the page to shift register between the editorial warmth of long-form explanation, the authority of headings, and the exactitude of protocol data.

Display — DM Serif Display
Transaction 0x006B
Login Request

Used for page titles, section headings, and hero text. The wedge serifs evoke railway timetable typography without cosplaying as one. Set at display sizes only; never below 1.44rem.

Body — Source Serif 4

All multi-byte integers are transmitted big-endian. No padding or alignment bytes are used: the fields follow one another exactly as listed. The wireframe adapter automatically detects clients that XOR-encode text fields. Once detected, inbound payloads are decoded and outbound replies are encoded to match the client, without any additional configuration.

The editorial voice. Variable optical sizing provides comfortable reading at body sizes. Set between 0.833rem and 1.2rem for prose. Line height 1.7 for body copy, tighter for larger sizes.

Mono — JetBrains Mono
Flags: 0x00  Is-reply: 0x00  Type: 0x006B
ID: 0x00000001  Error: 0x00000000
Total-size: 0x00000024  Data-size: 0x00000024

The precise voice. Used for code, protocol data, field labels, badges, and UI chrome. Also serves as the small-caps substitute for section labels and navigation. Never exceeds 1rem in running text.

04 — Code Surfaces

Terminal Windows

Code blocks and protocol examples use the Kohl background with the traffic-light dots as window chrome. Syntax colours draw from the main palette rather than a generic IDE theme, reinforcing the design language even in technical contexts.

src/wireframe/preamble.rs 
// Validate the Hotline handshake preamble
pub fn validate(preamble: &HotlinePreamble) -> Result<(), HandshakeError> {
    if preamble.protocol_id != b"TRTP" {
        return Err(HandshakeError::InvalidProtocol);
    }
    if preamble.version != 0x0001 {
        return Err(HandshakeError::UnsupportedVersion);
    }
    Ok(())
}

06 — Texture

Zellij Patterns

The background pattern is a nested diamond motif derived from Moroccan zellij tilework. It renders as an inline SVG at very low opacity, providing depth without distraction. The pattern uses the Terracotta stroke colour and appears on alternating sections to establish visual rhythm.

The pattern also carries a second reading: rotated 45°, each diamond is a byte boundary; the nested layers echo the frame-within-frame structure of Hotline's multi-fragment reassembly. This dual interpretation — decorative and structural — is the essence of the design language.

Application rules: The zellij pattern appears only on Parchment or Sandstone backgrounds, never on dark surfaces. Opacity must not exceed 0.12. The pattern tile is 60×60px for section backgrounds and 40×40px for dividers and smaller elements.

07 — Motion & Accent

The Express Line

A 4px gradient bar in Terracotta → Saffron → Indigo appears at the top of the hero and the bottom of the footer. On the hero, it uses a slow CSS background-position animation (8 seconds, ease-in-out) to shimmer — evoking the movement of a train through landscape. This is the only persistent animation; all other motion is interaction-triggered.

Interactive transitions use cubic-bezier(0.22, 1, 0.36, 1) — a slightly overshooting ease-out that gives hover effects a physical quality without bouncing.
